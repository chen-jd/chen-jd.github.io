InputStream是一个抽象类，有抽象方法
abstract int read()
这个方法将读入一个字节，并返回读入的字节，或者在输入远结尾返回-1。
设计输入流时，必须覆盖这个方法，以提供适用的功能。
比如：
1.FileInputStram中的该方法从文件中读取一个字节，并返回这个字节。
2.System.in中的该方法从键盘中读取一个字节，并返回。

同样的，OutputSteam中也定义了
abstract void write(int b)
它向某个输出位置输出一个字节。

文件流：FileInputStram/FileOutputSteam,FileReader/FileWriter
这四个类是专门操作文件流的，用法高度相似，区别在于前面两个是操作字节流，后面两个是操作字符流。它们都会直接操作文件流，直接与OS底层交互。因此他们也被称为节点流。
用完的时候，记得手动关闭流。
对文件的操作是堵塞的。（可以使用RandomAccessFile解决）

缓冲流：BufferedReader/BufferedWriter ， BufferedInputStream/BufferedOutputStream
经过Buffered处理过的输入流将会从一个buffer内存区域读取数据，本地API只会在buffer空了之后才会被调用（可能一次调用会填充很多数据进buffer）。

经过Buffered处理过的输出流将会把数据写入到buffer中，本地API只会在buffer满了之后才会被调用。
使用Buffered，每次处理IO，不需要直接操作操作系统底层，会让IO操作效率更高。

BufferedReader提供readLine()方法能方便地读取一行数据。因此也成为行读取器。

RandomAccessFile实现文件随机读写。实现零内存操作，与普通io流相比，支持任意访问的方式，无需把文件从头读到尾。
RandomAccessFile是线程不安全的，可以使用多线程对同一文件进行操作。
RandomAccessFile通过指针的方式来访问文件。
|方法名|作用|
|—–|——-|
|getFilePointer()|返回记录指针的当前位置|
|seek(Long pos)|将文件记录指针定位到指定位置|

使用RandomAccessFile打开文件：

r代表只读的方式打开文件。
rw代表读写的方式打开文件，并且使用buffer，对文件的修改不会立即生效，只有缓冲区满了，或者流关闭才会更新底层存储设备。
rws代表读写的方式打开文件，并对内容和元数据都同步写入到底层存储设备。
rwd代表读写方式打开，对文件内容的更新同步更新至底层存储设备
(在搜索”rws”,”rwd”的时候百度全网都是上面的介绍各种复制。)
stackoverflow中的回答：

rws flushes the contents of the file and the modification date of the file.
rwd flushs the contents of the file, but the modification date might not change until the file is closed.
rw only flushes when you tell it to and doesn’t change the modifcation date until you close the file.
BTW rwd is much slower for writes than rw, and rws is slower again.

区别在于文件修改信息的更新时间。
